✗ AVANT — Sans NetDevOps
Tu reçois un ticket : ajouter VLAN 100 sur 12 switches. Tu ouvres Putty sur chaque équipement, tu tapes les commandes à la main à 2h du matin. Un switch mal configuré → coupure de production.
✓ APRÈS — Avec NetDevOps
Tu donnes l'instruction à Claude Code. Un script Netmiko se connecte aux 12 switches en parallèle, applique la même config vérifiée, génère un rapport. Zéro faute de frappe. Zéro oubli.
⏱ ~8 min Fondamental Pipeline CI/CD
DevOps
Philosophie qui unit développeurs et opérationnels. L'objectif : automatiser tout ce qui est répétitif pour livrer plus vite et sans erreurs humaines.
NetDevOps
DevOps appliqué aux réseaux. Au lieu de configurer manuellement chaque Cisco ou FortiGate, tu écris du code qui le fait, de façon reproductible.
CI/CD
Continuous Integration / Delivery. À chaque modification du code, des vérifications automatiques s'exécutent. Si tout passe → livraison automatique.
Pipeline
La chaîne de montage complète. Chaque outil fait son travail et passe le résultat au suivant, sans intervention manuelle entre les étapes.

Imagine une cuisine de restaurant. Sans DevOps : chaque cuisinier improvise sa recette. Avec DevOps : recettes standardisées, étapes dans l'ordre, qualité identique à chaque service. Le NetDevOps, c'est la même chose pour configurer un switch Cisco à 2h du matin sans faire d'erreur.

Nuit de maintenance chez un opérateur télécom : 3 switches Cisco à reconfigurer avant 6h00. Sans NetDevOps, un technicien passe 2h en SSH manuel, risque de faute de frappe à 3h du matin. Avec le pipeline : Claude.ai rédige la logique, Claude Code génère le script, Netmiko configure les 3 switches en 47 secondes. Le technicien supervise depuis son bureau — zéro erreur de frappe.

⏱ ~8 min Git Branches Versioning
Repository (repo)
Un dossier versionné sur GitHub. Chaque modification est enregistrée avec son auteur, sa date et son message.
Branche
Une copie isolée du code. Claude Code crée toujours une branche claude/<tâche>-<id> par tâche.
Commit
Un instantané du code à un moment donné. Chaque commit a un identifiant unique (ex: 4753287).
Pull Request (PR)
Une demande de fusion de branche vers main. GitHub Actions valide puis fusionne automatiquement si tout passe.

Imagine un classeur de chantier. Main = le classeur officiel. Chaque branche = une feuille de brouillon numérotée. Tu travailles sur le brouillon, tu le fais valider, et seulement ensuite tu l'intègres au classeur officiel.

Migration VLAN sur un switch de distribution : la branche claude/migrate-vlan-200-a3f1 contient 3 fichiers modifiés. GitHub Actions vérifie la syntaxe IOS avant le merge. Un collègue détecte une erreur dans la PR → tu corriges sur la branche sans toucher main. Si la migration pose problème en prod : git revert annule en 30 secondes, l'historique complet est conservé.

📐 RÈGLES BRANCHES Nom : claude/<tâche>-<id>
— Jamais réutiliser une branche déjà mergée
— Jamais pusher directement sur main
— Une tâche = une branche = une PR
// SIMULATEUR DE BRANCHES GIT
🧠 Question — Pourquoi ne jamais pousser directement sur main ?
⚠️ PIÈGE — LA BRANCHE RECYCLÉERéutiliser une branche déjà mergée pour une nouvelle tâche mélange l'historique git, rend les PR illisibles et peut déclencher la CI sur le mauvais contexte. Une tâche = une branche fraîche, toujours.
// CHEATSHEET — MODULE 03
git checkout -bCréer une branche : git checkout -b claude/<tâche>-<id>
git push -u originPousser et lier la branche distante : git push -u origin <branche>
Pull Request (PR)Demande de fusion branche → main, validée par GitHub Actions
mainVersion de référence, jamais modifiée directement
git revert <commit>Annuler un commit sans réécrire l'historique

Déploiement réseau
Comment le code généré par l'IA finit par arriver sur un switch Cisco.
// OBJECTIFS DU MODULE
  • Configurer un équipement réseau avec Python + Netmiko
  • Comprendre le rôle de WSL dans le pipeline
  • Utiliser correctement les placeholders <IP-HERE> et <PASSWORD-HERE>
⏱ ~10 min Python Netmiko WSL
WSL
Windows Subsystem for Linux. Fait tourner un vrai Linux à l'intérieur de Windows. Claude Code s'exécute dedans — pas dans PowerShell, pas dans VS Code.
Python
Le langage qui orchestre tout. Il appelle Netmiko, gère les erreurs, génère les rapports. Simple à lire, puissant pour l'automatisation réseau.
Netmiko
Librairie Python qui simule un technicien SSH sur un équipement réseau. Supporte Cisco, Fortinet, MikroTik, Juniper, Aruba...
Placeholders
Les secrets ne sont jamais écrits en clair dans le code : <PASSWORD-HERE> <IP-HERE>

Netmiko c'est un technicien virtuel qui se connecte au switch en SSH, tape les commandes que tu lui donnes, et te rapporte ce qui s'est passé. Sauf qu'il ne fait jamais de fautes de frappe et peut configurer 50 équipements en même temps.

30 FortiGate à mettre à jour simultanément après une vulnérabilité CVE : un script Python Netmiko itère sur la liste des IPs, se connecte en SSH, applique le patch et génère un rapport succès/échec. Durée : 4 minutes pour les 30 firewalls. Version manuelle estimée : 6 heures, avec risque d'erreur humaine à chaque étape.

WSL — Ubuntu 22.04
from netmiko import ConnectHandler device = { 'device_type': 'cisco_ios', 'host': '<IP-HERE>', 'username': 'admin', 'password': '<PASSWORD-HERE>', } with ConnectHandler(**device) as conn: output = conn.send_command('show version') print(output)

⏱ ~9 min Plugins Skills Optimisation
// PLUGINS OFFICIELS ANTHROPIC
/batch
Décompose une tâche en 5 à 30 unités indépendantes, crée un worktree git isolé pour chacune, les exécute en parallèle et ouvre les PRs automatiquement. Idéal pour appliquer une modification sur plusieurs fichiers ou repos simultanément.
/simplify
Lance 3 agents de review en parallèle sur tes fichiers modifiés. Chaque agent analyse sous un angle différent et détecte en moyenne 3 à 5 vrais problèmes par branche — avant le commit.
/code-review
4 agents indépendants : 2 vérifient la conformité au CLAUDE.md, 1 scanne les bugs, 1 audite la sécurité. La porte finale avant chaque merge — gratuit et disponible sans restriction de plan.
skill-creator
Crée des skills personnalisées depuis zéro. Tu décris ce que tu veux automatiser, il génère le SKILL.md complet prêt à déposer dans ~/.config/claude-code/skills/

/batch c'est embaucher une équipe de développeurs junior qui travaillent en parallèle sur des tâches indépendantes. /simplify c'est le tech lead qui relit avant de pousser. /code-review c'est l'audit final avant la mise en production.

Refactoring de 12 scripts Netmiko suite à un changement d'API Cisco : /batch découpe le travail en 12 sous-tâches parallèles, un worktree git isolé par fichier. /simplify relit chaque modification avant commit. /code-review audite la sécurité globale avant le merge. Résultat : 12 fichiers mis à jour en une session, sans régression, sans toucher main.

// SKILLS COMMUNAUTAIRES AVANCÉES
root-cause-tracing
Trace les erreurs profondes dans les scripts Python/Netmiko. Remonte la chaîne d'exécution jusqu'au déclencheur original — indispensable pour débugger un déploiement réseau qui échoue silencieusement.
using-git-worktrees
Crée des worktrees git isolés avec vérification de sécurité. Permet de travailler sur plusieurs branches simultanément sans risque de pollution de contexte entre les tâches.
test-driven-development
Impose d'écrire les tests avant le code. À utiliser systématiquement avant de coder les scripts de déploiement Netmiko — garantit que chaque commande réseau est vérifiable.
kaizen
Méthodologie d'amélioration continue inspirée du Lean japonais. Applique des cycles courts d'analyse et d'optimisation — cohérent avec l'approche NetDevOps itérative.
